# 避難收容所資料審計報告

資料來源：data.gov.tw/dataset/73242  
總筆數：5,973  

---

## 一、資料品質問題

### 1. 空值問題

**發現問題**：
- 村里缺失 209 筆 (3.5%)
- 地址缺失 3 筆 (0.05%)
- 災害類別缺失 184 筆 (3.1%)

**影響分析**：
- 缺失村里資訊影響鄉鎮級避難規劃
- 地址缺失降低避難所可及性
- 災害類別缺失影響風險評估

**處理方式**：
- 標記為 NA，保留原始資料完整性
- 不自動填補避免錯誤推論
- 建議後續人工確認或使用地理編碼補充

---

### 2. 經緯度異常

**發現問題**：
- 3 筆經度為 0 (明顯錯誤)
- 多筆緯度超出台灣合理範圍 (21°-26°N)
- 部分座標精度不足，僅整數值

**技術分析**：
```python
# 異常座標範例
異常記錄1: 經度=0, 緯度=24.5 (無效座標)
異常記錄2: 經度=120.5, 緯度=18.0 (超出台灣範圍)
異常記錄3: 經度=121, 緯度=25 (精度不足)
```

**處理方式**：
- 將異常值設為 NA
- 移除3筆經度為0的記錄
- 後續使用地址重新定位

---

### 3. 重複資料

**發現問題**：
- 305 筆重複座標 (5.1%)
- 部分為同一設施不同名稱
- 部分為座標精度不足造成假性重複

**重複類型分析**：
1. **真實重複**：同一避難所多筆記錄
2. **座標重複**：不同地點相同座標(精度問題)
3. **設施重複**：學校與活動中心位於相同地址

**處理方式**：
- 不直接刪除，保留所有記錄
- 新增 duplicate_flag 欄位標記重複狀態
- 建議人工確認重複原因

---

### 4. 電話格式混亂

**發現問題**：
- 科學記號：4.72E+11, 2.66E+15
- 格式不一致：02-12345678 vs 0212345678
- 特殊字元：#分機符號混亂

**具體案例**：
```
原始格式1: 4.72E+11 → 修復後: 472000000000
原始格式2: 02-29112281#1209 → 修復後: 0229112281
原始格式3: (089)731633 → 修復後: 089731633
```

**處理方式**：
- 統一轉為純數字格式
- 保留區域碼完整性
- 移除特殊符號

---

### 5. 字元亂碼

**發現問題**：
- 問號(?)替代中文字元
- 部分特殊字元顯示異常
- 疑似 UTF-8 / Big5 編碼問題

**亂碼類型**：
1. **文字替代**：?榔里 → 永榔里
2. **編碼錯碼錯誤**：磚?里 → 磚磘里
3. **符號異常**：E+11 科學記號

**處理方式**：
- 重新以 Big5 編碼讀取原始檔案
- 手動修正明顯亂碼
- 建立字元對應表
