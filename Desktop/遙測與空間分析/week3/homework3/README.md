# ARIA：台灣緊急避難所洪災風險評估系統

## 專案概述

ARIA（Assessment of River-Inundation Areas，河川淹水範圍評估）是一套全面的洪災風險評估系統，評估台灣各地緊急避難所面臨河川洪水的脆弱性。此專案超越基礎空間分析，為應急管理與撤離規劃提供可執行的洞察。

## 核心創新

- **多級風險分區**：三級緩衝區（500公尺、1公里、2公里），取代單一緩衝區分析
- **收容量缺口分析**：回答「安全避難所是否足夠？」，而不僅是「哪些避難所有風險？」
- **鄉鎮市區層級評估**：區域分析以利資源精準配置
- **互動式決策支援**：即時地圖與詳細避難所資訊

## 資料來源

1. **水利署河川面資料**
   - 直接 API 存取：`https://gic.wra.gov.tw/Gis/gic/API/Google/DownLoad.aspx?fname=RIVERPOLY&filetype=SHP`
   - 座標系統：EPSG:3826（TWD97）

2. **消防署避難收容所資料**
   - 來源：[data.gov.tw/dataset/73242](https://data.gov.tw/dataset/73242)
   - 包含：避難所位置、收容人數、聯絡資訊

3. **國土測繪中心鄉鎮界線**
   - 用於區域分析與行政區報告

## AI 診斷日誌

### 遇到的挑戰與解決方案

#### 1. 資料編碼問題
**問題**：data.gov.tw 的避難所 CSV 使用 Big5 編碼，造成 UnicodeDecodeError
**解決方案**：在 Notebook 中實作回退編碼偵測（UTF-8 → Big5）

#### 2. 座標驗證
**問題**：原始避難所資料包含無效座標（0, 0）與經緯度反置
**解決方案**：加入全面的座標清理：
- 經度範圍：119-122°
- 緯度範圍：21-26°
- 移除空值與零座標
- 保留資料清理統計

#### 3. CRS 一致性
**問題**：不同資料來源使用不同座標系統
**解決方案**：將所有資料標準化為 EPSG:3826（TWD97）以確保距離計算準確

#### 4. 多級緩衝區重疊
**問題**：位於多個緩衝區的避難所需要指定最高風險等級
**解決方案**：實作階層式空間連接邏輯：
- 高風險（500公尺）優先
- 中風險（1公里）排除高風險避難所
- 低風險（2公里）排除更高風險避難所

#### 5. 鄉鎮名稱欄位偵測
**問題**：不同資料集使用不同欄位命名慣例
**解決方案**：動態偵測鄉鎮名稱欄位（支援中文/英文變化）

#### 6. 記憶體管理
**問題**：大型資料集造成效能問題
**解決方案**：使用溶解河川幾何與高效空間運算

#### 8. KeyError: 'risk_level' 
**問題**：鄉鎮級分析時出現 KeyError，風險等級欄位未正確傳遞
**解決方案**：修改空間連接明確包含 risk_level 欄位
**程式碼修復**：在 `gpd.sjoin()` 中加入欄位選擇

#### 9. 所有避難所被標記為 'safe'
**問題**：緩衝區分析失敗，無法正確分配風險等級
**根本原因**：資料載入不完整和缺少座標驗證
**解決方案**：實作適當的資料清理和座標驗證

#### 10. Unicode 編碼問題
**問題**：Windows 控制台無法顯示 Unicode 字元和 emoji
**解決方案**：移除特殊 Unicode 字元並使用 ASCII 替代方案
**程式碼修復**：用純文字等價字元替換 emoji

#### 11. 檔案路徑編碼問題
**問題**：中文檔名在 Windows 中造成編碼錯誤
**解決方案**：使用適當的編碼處理和回退機制
**程式碼修復**：新增多種編碼嘗試（utf-8, big5, cp950）

### 技術決策

1. **緩衝距離選擇**：基於台灣地形與洪水撤離標準的 500m/1km/2km
2. **風險等級階層**：遵循應急管理協定（高 → 中 → 低 → 安全）
3. **收容量缺口指標**：風險收容比 =（高 + 中風險收容量）/ 總收容量
4. **視覺化策略**：遵循標準應急管理慣例的顏色編碼風險等級

### 效能優化

- 緩衝前溶解河川幾何以降低計算複雜度
- 對座標驗證使用向量化運算
- 實作適當索引的高效空間連接
- 優化叢集標記的互動式地圖渲染
